# IR v1.2：回旋飞回 pattern + 双 Bullet 接力实现

## 决策
✅ **入库**（设计 + 实现 + 闭环验证全部完成）

## 用户原始需求
> "类似 LOL 泰隆大招，发射一圈子弹到自身周围，悬停之后追着主角飞回，在发出去的时候没有伤害，追回的时候才有伤害"

## M0 关键发现（先验研究）

### 1) FlyType / TracePathType 枚举语义（[common.nothotfix.cs](../../../Assets/Scripts/TableDR_CS/NotHotfix/Gen/common.nothotfix.cs#L7382-L7412)）

**TBulletFlyType**：
- 5 = 沿轨迹移动 (TRACE_PATH) — 配合 TracePathType
- 6 = 追踪目标坐标点 (TRACK_TARGET_POINT)

**TTracePathType**（FlyType=5 时的子轨迹）：
- 3 = 折返轨迹 (TURNBACK) — "飞出 + 悬停 + 折返"
- 4 = 静态轨迹 (STEADY) — 飞到某点静止

### 2) BulletConfig 只有 3 个生命周期钩子

```
BeforeBornSkillEffectExecuteInfo  — 生成前
AfterBornSkillEffectExecuteInfo   — 生成后整个生命周期
DieSkillEffectExecuteInfo         — 死亡时
```

**没有"折返开始时"等中间钩子**。

### 3) 项目内"飞出+回飞"现成实现都不是用户要的

| 样本 | 模式 | 与用户需求差距 |
|------|------|---------------|
| 30212005 木影旋刃 | 碰到造伤害+触发回飞 | 伤害发生在飞出阶段，**反**用户需求 |
| 22017101 蝴蝶妖回旋镖 | AfterBorn 全程挂碰撞 | 出去回来都造伤，不是分阶段 |
| 300102 通用回旋镖 | 不用 BulletConfig，靠 ADD_FORCE 模拟 | 与子弹系统无关 |

**结论**：项目无现成实现，必须自己造。

## 设计方案 — 双 Bullet 接力

```
Bullet1（飞出+悬停）：
  FlyType=5 / TracePathType=4 (静态轨迹)
  AfterBorn = 空 ← 关键：飞出无碰撞 → 无伤害
  Die = ORDER → 在死亡位置创建 Bullet2

Bullet2（回飞）：
  FlyType=6 (TRACK_TARGET_POINT) 追施法者位置
  Speed = boomerang.return_speed
  AfterBorn = 子弹通用-碰撞模板 ← 关键：回飞才挂碰撞 → 有伤害
```

### IR 表达（v1.2）

```yaml
- bullet:
    pattern: 回旋飞回
    angles: [0, 45, 90, 135, 180, 225, 270, 315]
    boomerang:
      flight_frames: 30
      hover_frames: 20
      max_distance: 600
      return_speed: 2000
    hit:
      collision_radius: 100
      damage: { coef: 12000, type: 法术 }
```

## 关键技术决策

### 1) BulletConfig 内联 vs 表行
- **选内联**：在 SkillGraph JSON 内直接写 BulletConfigNode，不走 1SkillEditor.xlsx 表流程
- 复用 320146（30212005 木影旋刃）的字段模板 + 用 IR 参数覆盖 FlyType/Speed/MaxDistance/AfterBorn/Die 等关键字段
- 优点：不依赖客户端导表，策划自给自足

### 2) "各颗独立造伤"语义（用户决策）
- N 颗 Bullet2 都挂同一份碰撞模板 → 同目标可被多颗 Bullet2 同时打到造 N 份伤害
- 与 LOL 泰隆 R 原版一致

### 3) damage_phase=return_only 由结构自然实现
- 不需要 TSET_TURN_BACK_BULLET_ENTER_KEEP_STATE 等高级节点
- 双 Bullet 接力天然实现：Bullet1 不挂 AfterBorn / Bullet2 挂 AfterBorn

## 利

- **真做出 LOL 泰隆 R 模式**：项目里第一个真正的"飞出无伤 / 回飞才有伤"实现
- **IR 表达力大幅扩展**：不再受限于"直线子弹"单一模式
- **可复用**：之后任何"回旋类"需求一句 yaml 出（飞镖、回旋刀、引力球等）
- **GATE-0.5 + GATE-3.5 完整跑通**：先验研究 + 自审都按流程走，没踩 PostMortem #005 那些"AI 偷懒/健忘"的坑

## 弊

- BulletConfig 内联依赖 320146 字段模板：如果用户想完全独立的轨迹参数，TracePathParams 11 个参数语义需要进一步逆向
- 高级节点 step（M4）暂未实现：未来更复杂的子弹控制需求（如"飞出途中碰人不消失但记次数"）需要补 M4

## 噪音风险
低 — pattern=回旋飞回 是显式用户触发，不会误伤现有"直线子弹"用例

## 反思

**这次执行 vs PostMortem #005 的对照**：

| 失败模式 | #005 中表现 | 本次表现 |
|---------|-----------|---------|
| AI 偷懒（占位 ID） | 用 320032/200001 等捏造 | 严格走 GATE-0.5 用真实样本扫描 |
| AI 健忘（不穷举） | 时长规则只沉淀 3 条漏 1 条 | M0 主动逆向 BulletConfig 全字段、FlyType/TracePathType 全枚举 |
| 凭直觉设计 | explosion 加 QUICK_DAMAGE 冗余 | 双 Bullet 接力 + 反复验证拓扑前先读 30212005 真实样本 |

**结论**：GATE-0.5 + GATE-3.5 的强制流程**实质上把"AI 偷懒/健忘"的发生概率显著降低**。这次用户从需求提出到实测交付只用了一轮反馈（vs #005 的 4 轮）。
