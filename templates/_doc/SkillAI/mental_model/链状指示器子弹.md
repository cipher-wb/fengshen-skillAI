## 一句话本质

**端绑型 Beam 链状指示器子弹** = 用一个**空锚点子弹**（`Model=4`）+ **链特效**（`BulletConfig.ChainModel`）+ **双钩子**（`BeforeBornSE.FOLLOW_ENTITY` 绑头端 + `AfterBornSE.MODIFY_ATTR(118)` 绑尾端）实现的视觉连线 / 无伤害 / 引擎自动 LineRenderer。

链长 N 段 = N 个 BulletConfig 实例链式串联 / 每段 BeforeBornSE 各挂一对双钩子 / 自然 die 通过 60 帧（或 IP=N 段类比）超时控制 / 不需要显式 DESTROY_BULLET。

---

## 关键不变量 (candidate / B-DESIGNER-CHAIN-001 入库)

| # | 字段 | 值 | 语义 |
|---|------|---|------|
| 1 | `BulletConfig.Model` | **4** | 空锚点（不能填 0 / 必须 4 / 参见 memory `reference_empty_bullet_model.md`）|
| 2 | `BulletConfig.ChainModel` | 链特效 ID | 视觉来源 / 引擎自动 LineRenderer 渲染 |
| 3 | `BulletConfig.FlyType` | **0** | 静态不动（链段不飞 / 由两端 attr=118 端绑驱动）|
| 4 | `BeforeBornSE.FOLLOW_ENTITY.P[0]` | {V=75, PT=1} | 子弹自身（实体级引用）|
| 5 | `BeforeBornSE.FOLLOW_ENTITY.P[1]` | **{V=35, PT=5}** ⭐ | **施法者-根创建者**（穿透多层嵌套 / D-038 铁律）/ 或 V=37 怪物自释场景 |
| 6 | `AfterBornSE.MODIFY_ATTR.P[0]` | {V=75, PT=1} | 子弹自身 |
| 7 | `AfterBornSE.MODIFY_ATTR.P[1]` | {V=118, PT=0} | TBN_CHAIN_TARGET_ENTITY（链尾绑定目标 attr 编号）|
| 8 | `AfterBornSE.MODIFY_ATTR.P[2]` | {V=2, PT=5} | TCPT_TARGET_ENTITY（链尾 = 目标 / 通常上一段子弹尾）|
| 9 | `CREATE_BULLET.P[4]` 创建者 | V=35 或 V=37 | 与 FOLLOW_ENTITY P[1] 口径一致 |
| 10 | `CREATE_BULLET.P[10]` 伤害归属 | **V=41** | ORIGIN 系列 / 锁最初技能实例（不漂移）|

**双钩子协作机制**：

```
[上一节点(角色 / 上一段子弹尾)]
       │ FOLLOW_ENTITY (BeforeBornSE)
       ▼
[本段子弹 - 空锚点 Model=4]  ← ChainModel 渲染从 头端 到 尾端 的链特效
       │ MODIFY_ATTR(118) (AfterBornSE)
       ▼
[目标(下一节点 / 链尾)]
```

**与 D-038 的耦合（铁律根源）**：D-040 是"嵌套主体陷阱"在链状指示器场景的具体应用范式。违反 D-038（FOLLOW_ENTITY P[1] 填 V=1 主体）→ 链段头端绑到子弹自己头上而非主角 → 链断 / 0 视觉效果。

---

## 反直觉点

1. **Model=4 不是 0** — 空锚点专属 ID / 配链状指示器/纯视觉子弹时必填 4 / 见 memory `reference_empty_bullet_model.md`
2. **FlyType=0 链不飞** — 链段本身是静态的 / 视觉移动靠两端 attr=118 端绑驱动（引擎自动跟随）
3. **不需要 DESTROY_BULLET** — 60 帧自然 die 即可 / 加显式销毁反而增加复杂度
4. **MODIFY_ATTR(118) 是链子弹专属语义** — attr 编号 118 = TBN_CHAIN_TARGET_ENTITY / 普通子弹用不到 / 见 `BattleBulletAttrCollectComp.cs:123-129` 分支
