## 模板范式（candidate）

### M1. 对称兄弟模板范式（D-6106 / symmetric-sibling-template）

会心 146004835 与 化解 146004834 完全对称：

- 同 L2 调用 schema
- 同 PROBABILITY_EXECUTE 根
- 同 Tag41 写入模式（仅 cid 不同）
- 仅 attr 攻守方向相反
- 数学结构镜像

升 candidate 前需 ≥3 对对称对印证（闪避↔精准? 防御↔穿透?）

### M2. PROBABILITY_EXECUTE 概率分支根节点范式（D-6108）

- SkillEditor 概率事件用 **TSET_PROBABILITY_EXECUTE (SkillEffectType=27)** 而非 TSCT_RANDOM_RATE
- 节点本身管"成功/失败"两分支 / 概率值由子节点动态计算
- 与配子弹的 PROBABILITY 节点同源
- 升正式前需 grep TSET_PROBABILITY_EXECUTE 全工程引用统计 + C# enum 源码加固

### M3. L1 分流兄弟 L2 模板 TemplateParams ABI 对齐范式（D-6002）

L1 同一个分支节点下游的多个 L2 候选模板，必须 TemplateParams **数量+名称+顺序 100% 一致**（"ABI 对齐"），让上层一套参数无差异分流。

- 实证：146004930 vs 146004986 两 L2 模板 15 个 TemplateParams 完全一致 / 175000212 L1 调用时一套参数透传到两 L2 分支
- 阈值：≥3 个 L1 分流场景印证后升 candidate

### M4. 工具型 vs 流程模板范式（D-6110 / utility-template vs pipeline-template）

SkillEditor 模板按形态分两类（开放修饰 / 不排他）:

- **流程模板（pipeline-template）**：L0-L2 多步流程 / 节点数 ≥17 / 含 ORDER_EXECUTE + REPEAT + FIRE 等流程节点
- **工具型模板（utility-template）**：L3-L4 工具 / 节点数 ≤10 / 根 = CONDITION_EXECUTE / NUM_CALCULATE 等纯计算 / 多调用点参数一致 / 无副作用

典型工具型：146004987 取伤害类型（4 节点 / 被调 3 次参数一致）。

与 D-5401 NSC 模板族分类**正交**关系（不冲突 / 互补维度）。

### M5. NUM_CALCULATE 即根模板形态（D-6111）

模板根节点**不一定**是 ORDER_EXECUTE / 也可以是 **NUM_CALCULATE**（如 146004907 返回数值给主调方）。

少见但合法的模板根形态。升正式前需更多 NUM_CALCULATE 即根实证。

### M6. 单对 REG/UNREG 围绕业务节点 idiom（D-6003）

`REGISTER_SKILL_EVENT + 业务节点 + UNREGISTER_SKILL_EVENT` 三件套形成临时事件钩子窗口期。围绕 DAMAGE 节点使用时，BUFF/被动技能可在伤害触发链内动态累加增伤数值（如 Tag184 会心增伤累加 / Tag191 化解增伤累加）。

与 SkillEvent系统.md §UNREG 反复模式（已升正式 N 对 REG/UNREG ≥5 范式）**互补维度** / NOT 撤回。

### M7. 孤岛节点：被 group 包裹但未被 root ORDER 引用 ≠ 生效（D-6004 / 高价值 / watching）

SkillEditor 模板内存在"被 group 包裹但未被 root ORDER.Params 引用"的孤立节点群——是模板演化/分工剥离时未清理的接口对齐遗物。审核模板时不能假定"group 内的节点都生效"，必须沿 root ORDER 追溯实际引用链。

- 实证：146004986 真实伤害模板 17 节点 / 生效仅 10 节点 / 4-6 个 group 内节点未被执行
- 与 PostMortem #037 关系：第二种"节点存在但不生效"模式
  - 已知：顶层 nodes ≠ refs 导致不渲染（PostMortem #037）
  - 新模式（本 delta）：root ORDER 不引用导致不执行
- lint 工具建议：未来 `skill_lint.py` 加"孤岛节点检测"规则

## 子弹序号 → 威力 Tag 配对速查（v0.16.41 / 2026-05-13 memory 升级入库）

4. **通用伤害模板** `SkillGraph_【模板】子弹通用逻辑-伤害.json`（根 190016485）根据"本子弹序号"参数（1~5）决定 Tag ID

| 3 号 | **1202** | **1212** |
| 4 号 | 待补 | 待补 |
| 5 号 | 待补 | 待补 |

**配置位置**：首节点 `SkillConfigNode` → 字段 `SkillParamList` → 每条 entry `{Key: TagID, Value: 数值, Description: "..."}`。数值 = 万分比（6000 = 60%）。
