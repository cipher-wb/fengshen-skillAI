## 一句话本质

**项目所有伤害（含近战）都通过"子弹"造成**——这是项目战斗框架的统一抽象。`BulletConfig` 是子弹蓝图配置容器，提供 3 个**一次性**生命周期钩子 + Model 字段控制视觉。要实现"每帧逻辑"必须用 `REPEAT_EXECUTE` 包装。

---

## 关键不变量（违反就 100% 出问题）

### A. BulletConfig 三个生命周期钩子都是「一次性调用」

> 来源：[memory/feedback_bulletconfig_afterborn_repeat.md](../../../memory/feedback_bulletconfig_afterborn_repeat.md) / PostMortem #032 / 2026-05-11 PoC 30212018 立法。

| 钩子字段 | 语义 | 调用次数 |
|---------|------|---------|
| `BeforeBornSkillEffectExecuteInfo` | 子弹出生前 | **一次性** |
| **`AfterBornSkillEffectExecuteInfo`** | **子弹出生后** | **一次性 ⚠️ 最常错（误认为每帧）** |
| `DieSkillEffectExecuteInfo` | 子弹死亡时 | **一次性** |

**反直觉点**：AfterBorn 字面上像"出生之后持续"，实际是"出生瞬间触发一次然后结束"。要做每帧逻辑（旋转 / 加速 / 移动）**必须**用 `TSET_REPEAT_EXECUTE` 节点包装。

### B. OnTick 标准链结构（必须照抄 30212017 风车）

```
BulletConfig.AfterBornSkillEffectExecuteInfo
  → ORDER (出生后一次性入口)
      ├─ P[0] = REPEAT_EXECUTE (interval=1帧, count=1000)
      │           body = OnTick body ORDER (旋转 + 加速 + 移动)
      └─ P[1..N] = 其他出生时 setup (碰撞模板调用 / 注册 callback / ...)
```

**4 层结构**（缺一不可）：
1. `BulletConfig` 容器
2. `BulletConfig.AfterBorn → ORDER` 入口
3. `ORDER.Params[0] → REPEAT_EXECUTE` 包装层
4. `REPEAT.body → OnTick body ORDER` 实际每帧执行体

⚠️ **REPEAT 的 interval / count 不能 0 / -1**（[PostMortem #026](../postmortem/2026-05-09-026-repeat-execute-c++-cap-runtime-error.md) C++ crash）。

### C. 删「孤立 REPEAT」前必须三方向 grep 引用

REPEAT 节点表面"无 incoming refs"是错觉——它通常被三种字段引用：

1. 某 `ORDER.Params[]` 子项
2. `BulletConfig.{Before/After/Die}BornSkillEffectExecuteInfo` 直接指向
3. `SkillConfig.SkillEffectExecuteInfo` 顶层指向

**任一方向 hit → 不能删**。删之前用三个 grep:

```bash
# 方向 1：被 ORDER.Params 引用？
rg "\"Value\":\s*<REPEAT_GUID>" SkillGraph_*.json

# 方向 2：被 BulletConfig 引用？
rg "\"SkillEffectConfigID\":\s*<REPEAT_GUID>" SkillGraph_*.json

# 方向 3：被 SkillConfig 引用？
rg "\"SkillEffectConfigID\":\s*<REPEAT_GUID>" SkillGraph_*.json
```

### D. 空子弹 Model 必填 4（不要填 0）

> 来源：[memory/reference_empty_bullet_model.md](../../../memory/reference_empty_bullet_model.md) / 2026-05-08 用户口头确立 / PostMortem #040（待落盘）。

当创建"锚点空子弹" / "无视觉表现 BulletConfig" 时：

| Model 值 | 含义 | 后果 |
|---------|------|------|
| **4** | ✅ 项目专属"空模型 ID" | 不渲染 / 正常逻辑可用 |
| 0 | ❌ NULL 默认值 | 引擎走 fallback 路径 / 可能"弹不出来" / 不可预期 |

**典型场景：锚点空子弹**（capture-at-cast pattern）：

```json
{
  "ID": 320149,
  "Name": "千叶散华锚点空子弹",
  "Model": 4,          // ← 关键：4 = 空模型
  "Speed": 0,
  "LastTime": 60,
  "AfterBornSkillEffectExecuteInfo": {"SkillEffectConfigID": 32002513}
}
```

- 在 cast 瞬间创建 `Speed=0 / Model=4` 不可见锚点子弹
- 捕获 caster 当时的位置（不跟随后续位移）
- 真正的子弹通过 anchor 的 `AfterBornSkillEffectExecuteInfo` 链接发出
- 即使 caster 跑了，发射点仍是 cast 时位置（**位置捕获 idiom**）

### E. 子弹序号 → SkillTag ID 配对铁律

> 来源：[memory/reference_bullet_damage_tag_pairs.md](../../../memory/reference_bullet_damage_tag_pairs.md) / 用户 2026-05-09 拍板"每个技能要做审核"。完整主表见 [SkillTag系统.md §D 1 号子弹的伤害对子配对铁律](SkillTag系统.md) 与 [伤害管线.md](伤害管线.md)。

**核心规则**：
1. **所有技能伤害都通过子弹造成**（含近战，统一抽象）
2. **一个技能可造成多次伤害**（最多 5 号子弹，常用 1/2/3 号）
3. **每次伤害必须由"威力系数 + 额外伤害"成对组成**：威力系数 = 倍率 / 额外伤害 = 固定值
4. **通用伤害模板** `SkillGraph_【模板】子弹通用逻辑-伤害.json` 根据"本子弹序号"参数（1~5）决定 Tag ID

**配对表（出现 A → 必须出现 B）**：

| 子弹序号 | 技能威力系数 Tag ID | 技能额外伤害 Tag ID |
|---------|---------------------|---------------------|
| 1 号 | **3**（技能威力系数） | **5**（技能额外伤害） |
| 2 号 | **1201** | **1211** |
| 3 号 | **1202** | **1212** |
| 4 号 | 待补 | 待补 |
| 5 号 | 待补 | 待补 |

**配置位置**：首节点 `SkillConfigNode` → 字段 `SkillParamList` → 每条 entry `{Key: TagID, Value: 数值, Description: "..."}`。数值 = 万分比（6000 = 60%）。

## 反直觉点（容易踩的）

1. **"AfterBorn" 不是"出生后持续"** — 是"出生瞬间一次"。要每帧必须 REPEAT 包装。
2. **REPEAT 没有 incoming edges 不等于孤立** — 三方向 grep 必查。
3. **Model=0 ≠ 空子弹** — 必须用 Model=4 才是"项目约定的空模型"。
4. **子弹脱离施法者** — 伤害归属字段（CREATE_BULLET P[10] = V=41 ORIGIN）跨嵌套不漂移；嵌套 BornSE 内"主体 V=1" 指子弹自身（见 [参数与上下文.md §嵌套主体语义陷阱](参数与上下文.md)）。
5. **不能共享 BeforeBorn / AfterBorn 入口** — 配两种子弹（强化/普通）时必须完整复制链路（[SkillEditor文件结构.md §unique-parent](SkillEditor文件结构.md)）。
