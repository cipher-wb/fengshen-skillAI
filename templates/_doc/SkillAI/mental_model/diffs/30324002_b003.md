## 关键发现（DIFF）

### 发现 1：CdType=0 字典语义需修订（盲点 1 解决）

**v0.3 字典写**："CdType=0 = 标签型 / 被动型 / 无 CD / 引擎托管"
**本样本真证**：CdType=0 + CdTime=0 + **主动技** + EnhanceSkillBuffConfigID=0 + 不是 SubType=0 标签型

→ **强证 v0.3 字典 §D 需修订**：
- "CdType=0 ≠ 标签型/被动型独占"
- 真实语义应是"**无内置 CD（引擎托管或自定义机制限频次）**"
- 本样本是主动技但 CdType=0 + CdTime=0 — 通过自身 SkillTag 蓄势机制限制释放频次（17 GET_SKILL_TAG + 3 MOD_TAG 强信号）

### 发现 2：ADD_BUFF + BuffConfigNode 不平衡是常态（盲点 4 二阶印证）

| ADD_BUFF rid | buff_id_param2 | flow 内有 BuffConfig 节点？ |
|------|------|--------------|
| 1023 | 2200035 | ✓ rid=1024 (2200035) |
| 1035 | 2200105 | ✓ rid=1104 (2200105) |
| 1038 | 2200026 | ✗ **无 flow 内 BuffConfig 节点** |
| 1043 | 2200026 | ✗ 同上（与 1038 引用同 buff） |

**关键**：4 ADD_BUFF / 2 BuffConfig 不平衡的真相不是"flow 引用外部表"，而是：
- **flow 内 BuffConfig 节点是"声明 + 元数据"**（含 BuffNameKey / EffectID / Icon 等编辑器侧字段）
- ADD_BUFF.Params[2] 是**全局 BuffConfig.xlsx 表 ID**（无论 flow 内有没有 BuffConfig 节点都填这个 ID）
- flow 内 BuffConfig 节点**可选**（声明本技能定义的 buff），不是必需
- 引用其他技能定义的 buff（如 2200026）只填 ID，不在本 flow 加 BuffConfigNode

→ v0.3 Buff 系统不变量 1 需修订："BuffConfigNode 是 buff 实体定义"应改为"BuffConfigNode 是 flow 内 buff 元数据声明（编辑器侧），ADD_BUFF.Params[2] 才是全局 BuffConfig 表 ID（运行时寻址）"

### 发现 3：ADD_BUFF 字段语义初步揭示

5 样本 ADD_BUFF Params 全部 7 项，前 3 项一致：
- **Params[0].Value** = 目标选择/属性？（30324002 全 2，30325001 mix 1/2/3，30531008 mix 3/35）
- **Params[1].Value** = 3（除 30535000 rid=1070 是 35）— **buff 类型？**
- **Params[2].Value** = **buff_config_id（全局表 ID）** ← 关键
- Params[3].Value = duration（多数 0 / 数百 / -1 永久）
- Params[5].Value = 1（多数）/ 实体 uid 字面量

→ ADD_BUFF Params 字段需新增到节点字典 / Buff 系统页
