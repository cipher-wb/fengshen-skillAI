## 认知盲点

1. **"凝壁"不是召唤实体，是带视觉的 AOE 子弹** — 4 个 bullet + 4 个 model 实现"墙体外观"的视觉效果，碰撞用 bullet 系统而非召唤系统。**召唤系统不在本样本触碰**，我对"凝壁=召唤"的猜想完全错。
   - **对预测策略的影响**：名字含"墙/壁/盾"≠ 召唤系统，可能只是 bullet 加上特殊 model。

2. **持续施法模板 146003779 是跨门派核心模板** — 30234006（火宗门）+ 30325002（土宗门）都用，**这是宗门"蓄力大招"的统一骨架**。识别这个模板 ID 是预测高级技能的关键。

3. **SkillIndicatorParamTagConfigIds 揭示了"指示器参数 tag-driven"机制** — 之前我只知道 SkillTagsList 是技能内部数值；现在发现**指示器外观（宽度/长度）也能挂 tag** ↔ 这意味着指示器在运行时是 tag-reactive 的，与"修炼等级提升 → 指示器变大"这种动态需求直接挂钩。
   - **对子系统的影响**：触发器与碰撞 / 指示器系统应有"指示器参数双源（直接值 vs tag 引用）"的不变量。

4. **REPEAT_EXECUTE 在神通技能里 4 次出现** — 配合已沉淀的 PostMortem #026（C++ 200 次硬上限），神通蓄力大招高度依赖 REPEAT_EXECUTE 实现"持续 N 帧重复触发"。

5. **跨技能 SkillTag 读取确实存在** — tag 301 / 1001 不在本技能任何声明里，且 SkillConfig 也没声明，**真正属于跨技能读模式**。Lint E019 在这种情况下是 true positive，但需要分清：
   - false positive：tag 在 SkillConfig.SkillTagsList/SkillDamageTagsList 已声明
   - true positive：tag 完全未声明 → 真跨技能读 (Pattern C, Param[1]=源技能ID 或 Param[1]=0/PT=0 实体级)
   - **lint 规则改进点：先检查 SkillConfig 顶层，再判 E019**

6. **CdTime: 900 帧（15秒）+ SectXinfaEnergyValue: 100** — 神通是消耗心法能量的长 CD 大招，与神通武学概念对齐。
