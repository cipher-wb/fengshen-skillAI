3. v0.3 SkillEvent 系统页应在 §字段语义补："`SkillEffectType` 顶层字段 = 节点类型 ID（REGISTER=68 / UNREGISTER=69）；不要与 Event ID 混淆"

### 发现 3：心法不必有 EnhanceSkillBuffConfigID

**预测错**：v0.2 / v0.3 暗示"心法 → EnhanceSkillBuffConfigID 反向引用"是常态
**真证**：30535000 SkillNameEditor 含"传承心法"+ Mode C 纯 Passive，**EnhanceID=0**

→ v0.3 Buff 系统不变量 2 ("`EnhanceSkillBuffConfigID` 是心法的硬信号") 需精化：**仅"替换型普攻心法"才用 EnhanceID（如 30531005=2250099）；标签效果型/被动型心法不必填**。

→ 心法判定的硬信号修订为：
- **替换型普攻心法**：CdType=4 + EnhanceID ≠ 0 + 文件名"心法/功法-普攻"（30531005）
- **标签效果型心法**：CdType=0 + Mode C 纯 Passive + 文件名"标签效果/传承心法"（30534004 / 30531006 / 30535000 / 30531008）
- **被动型功法**：CdType=0 + Mode C + 任意 EnhanceID（30535000 EnhanceID=0 也算）
