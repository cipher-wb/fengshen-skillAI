## 认知盲点

1. **文件名 ≠ 技能真名** — 30124001 文件名"飞星印_地阶"是编辑器侧的项目代号或历史命名，真实运行时 SkillNameEditor="地阶火球普攻"。**SkillConfig.SkillNameEditor 是 ground truth**。这影响所有"按名字猜功能"的预测准确度。
   - **对预测策略的影响**：闭卷 PREDICT 时不能只看文件名，还要先打开 SkillConfig 看真名。或者预测后立刻在 DIFF 阶段把"文件名 vs 真名"作为单独差异维度。

2. **连招模板 38000289 是普攻型骨架** — 像 146003779（持续施法）一样，38000289（连招2段）是另一个"超模板"。**至少存在两个家族的核心模板**：连招类（38xxxxxxx）vs 持续施法类（146xxxxxx），且模板 ID 段位语义化。

3. **SkillEffectPassiveExecuteInfo 是普攻的双引擎** — 主流程（SkillEffectExecuteInfo）+ 被动流程（PassiveExecuteInfo）同时存在，意味着"普攻命中后还会触发另一条被动 effect"。我之前从未把 Passive 当成独立 root。
   - **对子系统的影响**：mental_model 需要"技能入口（SkillEntry）"概念，澄清 Active root vs Passive root vs flow-internal RUN_TEMPL root。

4. **SkillTag 1201 + 3 是伤害对子标配** — 与 [memory/reference_bullet_damage_tag_pairs.md](memory/reference_bullet_damage_tag_pairs.md) 完全一致（1号子弹的 tag pair）。**这条铁律在真实样本里得到证实**，confidence 升级。

5. **概率分支（PROBABILITY_EXECUTE）是普攻随机化机制** — 4 次出现，可能是"普攻有 X% 概率触发额外特效"，这是个独立的概率系统。
