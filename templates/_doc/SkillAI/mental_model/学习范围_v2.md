## 3. picker 强制 enforcement 规则（curator 必须遵守）

### Rule 1：sub_category 字段必填 + 在白名单内

> **v2.1 housekeeping #4 修复（2026-05-11 / B-020 R0 揭出）**：原 BLACKLIST_PREFIXES 只匹配顶级"宗门技能/废弃"前缀 / 嵌套"宗门技能/宗门心法/废弃（老心法）/" 漏判 / 305920 + 3019311 + 303915 + 301903 + 303921 + 302925 + 1860231 等被误 WHITELIST_pass 通过。改用 "path 任意位置含 '废弃' → 拒" 通用规则。
>
> **v2.3 housekeeping #7 修复（2026-05-11 / 用户拍板 / v0.16.11 升正式）**：通用 BUFF 子目录大量"弃用"形态（B-024~B-032 累积观察 7 例 fs 真扫全集 / 100% 集中通用BUFF 子目录）/ picker_v2 v2.2 漏判候选 / 新增 "path 任意位置含 '弃用' → 拒" 通用规则（与 '废弃' 通用规则同源同规则）。

```python
def is_in_scope(path: str) -> bool:
    """白名单严过滤"""
    WHITELIST = [
        '宗门技能/木宗门技能/', '宗门技能/火宗门技能/',
        '宗门技能/金宗门技能/', '宗门技能/水宗门技能/',
        '宗门技能/土宗门技能/',
        '宗门技能/BD标签/', '宗门技能/宗门标签/',
        '宗门技能/通用BUFF/', '宗门技能/其他（有用）/',
        '宗门技能/宗门心法/',
        '技能模板/',
    ]
    # 黑名单优先（防误判）
    BLACKLIST_PREFIXES = ['宗门技能/废弃']  # 顶级前缀（保留作 fallback）

    # ⭐ v2.1 housekeeping #4 修复：嵌套黑名单通用规则（"废弃"）
    # path 任意位置含 '废弃' 路径段 → 拒（覆盖嵌套子目录 + 文件名含"废弃"形态）
    if '废弃' in path:
        return False

    # ⭐ v2.3 housekeeping #7 修复（2026-05-11 / 用户拍板）：嵌套黑名单通用规则（"弃用"）
    # path 任意位置含 '弃用' 路径段或文件名 → 拒（通用BUFF 子目录"通用BUFF_xxx_弃用.json"形态 /
    # 与 '废弃' 通用规则同源同规则）
    if '弃用' in path:
        return False

    for bl in BLACKLIST_PREFIXES:
        if path.startswith(bl):
            return False
    return any(path.startswith(wl) for wl in WHITELIST)
```

### Rule 2：picker.json 必带 in_scope_verdict 字段

每个 pick 必须含：
```json
{
  "skill_id": "...",
  "path": "宗门技能/木宗门技能/SkillGraph_xxxx.json",
  "in_scope_verdict": "WHITELIST_pass",  // 或 "BLACKLIST_reject_*"
  "in_scope_evidence": "path.startswith('宗门技能/木宗门技能/') == True"
}
```

### Rule 3：picker 选样前先 grep 本文件

curator picker 启动时第一件事是 grep 本文件取白名单，**不能凭"段位均衡"选样**。
段位（30xxxxxxx_8d / 30xxxx_6d 等技术分类）只作为**白名单内的分层指标**，不构成选样唯一标准。

### Rule 4：mental_model 不变量必须标 evidence_scope

每条 mental_model 不变量条目加 `evidence_scope` 字段：

- `evidence_scope: 范围内` — 证据 ≥3 例都来自白名单（高可信 / 实战配宗门技能直接引用）
- `evidence_scope: 跨范畴含老技能` — 证据混合白名单+黑名单（中可信 / 实战引用时注意可能是过时规律）
- `evidence_scope: 仅黑名单（历史样本）` — 证据全来自黑名单（低可信 / 仅作技术参考 / 实战配宗门技能不要直接套用）

### Rule 5：90 历史样本回溯标注

B-001~B-017 的 90 学过样本不重启 / 但 mental_model 各不变量条目必须按 Rule 4 回溯标 evidence_scope。
