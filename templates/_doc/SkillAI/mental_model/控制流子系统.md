## 一句话本质

控制流子系统是编辑器真正的"心智地基"——**控制原语家族（5 种）+ 时间锚点（CdType 3 种）**。整个技能 70%+ 的节点是 ORDER + DELAY + RUN_TEMPL 组成的"调度网"。控制流频次（语料统计 40,283 次）是子弹（2,398 次）的 17 倍——**真正的工作量在控制流而非子弹**（[learning_log §1.3](learning_log.md)）。

## 关键不变量（违反就 100% 出问题）

### A. 5 种控制原语家族

| 原语 | 节点 | 用途 | B-001 出现频率 |
|------|------|------|----------------|
| 顺序 | `TSET_ORDER_EXECUTE` | 串行执行多分支 | **5/5** 全部触碰，每样本 11~18 次 |
| 延迟 | `TSET_DELAY_EXECUTE` | 等待 N 帧后触发 | **5/5** 全部触碰，每样本 4~9 次 |
| 概率 | `TSET_PROBABILITY_EXECUTE` | 概率分支 | 30124001 出现 4 次 |
| 重复 | `TSET_REPEAT_EXECUTE` | 重复执行 N 次 | 30325002 出现 4 次 |
| 条件 | `TSET_CONDITION_EXECUTE` | 条件分支 | 多样本均有，与 `TSCT_VALUE_COMPARE` / `TSCT_AND` 配合 |

### B. ORDER_EXECUTE 是 dynamic port — 出边 outputPortIdentifier 必须 `"0"`

**PostMortem #001 铁律**：`TSET_ORDER_EXECUTE` 的所有出边 `outputPortIdentifier` 必须为字符串 `"0"`，不是 `"1"` `"2"` 这种序号（[postmortem #001](../postmortem/2026-05-08-001-dynamic-port-id.md)）。这是 dynamic port 节点的统一规则，违反会导致编辑器加载失败或运行时分支错乱。

### C. REPEAT_EXECUTE 间隔=0 时的硬上限

**PostMortem #026 C++ 引擎硬上限**：当 REPEAT 节点 `Interval=0` 时，`iExecuteCount ≤ 50` 安全 / `≤ 100` 极限。超过 100 会被 `BattleEffectFactory.cpp` 引擎层硬切断（[postmortem #026](../postmortem/2026-05-09-026-repeat-execute-limit.md)）。蓄力大招 / 持续施法严重依赖 REPEAT_EXECUTE，必须遵守。

### D. CdType 字典（决定整个时间轴，B-002 字典扩充）

| CdType | 语义 | 典型场景 | 样本 |
|--------|------|---------|-----------|
| **0** | **无内置 CD（引擎托管或自定义机制限频次）——不限被动型，主动技也可设 CdType=0** | 两类常见组合：(a) 主动技 CdType=0 + CdTime=0：靠 SkillTag 蓄势 / 自定义机制限频次（30214004 / 30324002）；(b) 标签效果 / 被动心法 CdType=0：引擎托管事件触发（30534004 / 30535000 / 30531008） | 30433001（水身法）/ 30534004（火心法蓄炎）/ 30214004（火多段连击+蓄力）/ 30324002（火焚骨流主动技）/ 30535000（土传承心法）/ 30531008（金标签效果-蓄势） |
| **1** | 即用即冷却（推测：立即 CD） | 奇术位移类 | 30215001（土奇术-奔岩突进） |
| **3** | 技能结束后 CD | 大招型 / 神通型 | 30234006（火大招）/ 30325002（土神通）/ 30325001（土天坠石牢-地阶-神通，CdTime=900） |
| **4** | 连招 | 普攻型 / 心法型（替换普攻） | 30124001（火普攻）/ 30531005（金心法）/ 30134001（火天阶功法-普攻，CdTime=21） |
| **5** | 连招 + 技能结束后 CD | 混合型 | 30431000（金身法） |

**CdType 决定整个技能时间轴的拓扑** — CdType=4 必有 ComboCdList；CdType=3 必长 CdTime；CdType=5 双段 CD 配置（[diffs/30531005.md](diffs/30531005.md) 教训 5）。

**CdType=0 不是"标签型/被动型专属"**（B-003 D-301 修订）：B-003 4 阶证据（30214004 hold-out + 30324002 主动技 + 30535000 心法被动 + 30531008 标签效果）共同表明 CdType=0 的真实语义是"无内置 CD（引擎托管或自定义机制限频次）"，主动技也可设 CdType=0，由 SkillTag 蓄势或自定义机制控制频次。CdType=1 仍 B-002 单样本观察，待 B-004+ 印证。

## 反直觉点（容易踩的）

1. **70%+ 节点是控制流而非业务节点** — 100+ 节点的技能里，bullet/buff/damage 业务节点其实少数，大量 ORDER + DELAY + RUN_TEMPL 是骨架。**真正的工作量在调度网**（[learning_log §1.3](learning_log.md)）。

2. **"蓄力技能"不是 ORDER + REPEAT 手写循环** — 看到蓄炎/蓄力/持续不要预测"层数累加器 + ORDER × N"，**实际是 RUN_TEMPL 调持续施法超模板**（[模板系统.md](模板系统.md)）。控制流被模板封装。

3. **PROBABILITY_EXECUTE 在普攻里 4 次** — 概率分支不只是"百分比触发特殊效果"，普攻类技能高度依赖 PROBABILITY 实现"随机化反馈"（30124001 4 次）。

4. **REPEAT_EXECUTE 蓄力技能 4 次出现** — 神通蓄力大招高度依赖 REPEAT_EXECUTE 实现"持续 N 帧重复触发"（30325002 4 次），必须配合 PostMortem #026 上限规则使用（[diffs/30325002.md](diffs/30325002.md) 教训 4）。

5. **DELAY 频次 ≠ corpus_scan top8 显示** — corpus_scan 截 top8，30325002 真实 DELAY=4 但 corpus 显示 0，30431000 真实 MOD_TAG=3 但 corpus 显示 0。**不能仅凭 corpus_scan 判断"无 DELAY"**（[batch_buffer/B-001_actionable.md A1](batch_buffer/B-001_actionable.md)）。

## 与其他子系统的因果关系

- 控制流 ←→ **模板系统**（包含关系）：超模板封装控制流，flow 顶层只剩单 RUN_TEMPL，控制流移到模板内部。
- 控制流 → **SkillTag 系统**：CONDITION_EXECUTE 读 tag（VALUE_COMPARE）决定分支走向；MOD_TAG 是控制流的副作用之一。
- 控制流 → **Buff 系统**：buff_layer 读出送入 VALUE_COMPARE / CONDITION_EXECUTE。
- 控制流 → **SkillEntry 系统**：Active 子树 / Passive 子树分别有自己的控制流网。
