# 技能节点字典

> **使用方法**：配技能时按节点名查表。每个节点列出：用途、Params 表（按下标）、引用类型、易错点。
> **权威源**：`Assets/Thirds/NodeEditor/0_说明文档/*.csv` + `Assets/Thirds/NodeEditor/Nodes/*.Custom.cs` 双向交叉验证。
> **注释格式**：`Params[N] = 字段名 (类型/枚举)：含义`。"类型"可能是 `int` / `Frame`（帧数）/ `‱`（万分比）/ `<TableName>` ID 引用 / `<EnumName>` 枚举值 等。

## 概述

本字典覆盖：
- 6 个主节点（BaseConfig）
- 26 个有 .Custom.cs 定义的效果节点 + 30+ 高频效果节点（不全的部分通过 CSV 文档第 1 份补充查询）
- 2 个有 .Custom.cs 定义的筛选节点 + 22 种筛选类型
- 4 个有 .Custom.cs 定义的条件节点（条件类型共约 95 种，本文档覆盖核心 4 类）

未列出的节点请直接查 `0_说明文档/1_战斗-效果节点配置.csv`（559 行，按中文名顺序排列）。

## 1. 主节点（BaseConfig）

### 1.1 SkillConfigNode（技能根）

**对应表**：`SkillConfig`。一图一个，是技能蓝图的入口。

**关键字段分组**：

| 分组 | 字段 | 含义 |
|------|------|------|
| 标识 | `ID` / `SkillNameKey` / `SkillDescKey` / `Icon` | 技能 ID / 多语言文本 ID / 图标 |
| 编辑器辅助 | `SkillNameEditor` / `SkillDescEditor` | 编辑器内显示用文本（保存时自动补"。"） |
| 元素 | `ElementType` (TElementsType) | 五行 1金/2木/3水/4火/5土 |
| 类型 | `SkillMainType` (TBattleSkillMainType) / `SkillSubType` (TBattleSkillSubType) | 主类型：招式 ZHAO_SHI / 公攻法宝 / 防御法宝 / 心法 等 |
| 标签 | `FeatureLabel` / `BDLabels` / `SkillAITags` | 功能标签 / 绑定标签 / AI 标签 |
| 伤害 | `DamageType` | 物理 / 法术 / 真伤 |
| 消耗 | `MPCost` / `HunLiValue` / `SectXinfaEnergyValue` | 法力 / 魂力 / 心法能量 |
| 时序 | `SkillCastFrame` (Frame) / `SkillBufferStartFrame` / `SkillBufferFrame` / `SkillBaseDuration` | 前摇 ≤ 缓冲区起始 ≤ 缓冲区时长 ≤ 基础时长 |
| 时序-移动 | `IsSkillBufferFrameCanMove` / `SkillCastIsNotInterruptable` | 缓冲区可移动？/ 前摇不可打断？ |
| CD | `CdType` (TSkillColdType) / `CdTime` (Frame) / `ComboCdList[]` / `CDMaxStoreCount` / `SkillFixCdTime` | CD 类型（常规/连招/技能结束）/ 主 CD / 连招每段独立 CD / CD 蓄满次数 |
| 范围 | `SkillRange` / `SkillMinRange` / `AISkillRange` / `ExtraAlertRange` | 主释放范围 / 最小释放范围 / AI 范围 / 额外警示范围 |
| 范围-动态 | `SkillRangeTagConfigId` / `SkillMinRangeTagConfigId` | → SkillTagsConfigNode（运行时按 Tag 取值） |
| 入口 | `SkillEffectExecuteInfo {SelectConfigID, SkillEffectConfigID}` | 主动释放入口（筛选 + 效果） |
| 入口-被动 | `SkillEffectPassiveExecuteInfo {SelectConfigID, SkillEffectConfigID}` | 被动/装备时入口 |
| 入口-卸下 | `SkillEffectOnUnEquip` | 卸下时执行 |
| 入口-打断 | `SkillEffectOnSkillCastInterrupt` | 释放被打断时执行 |
| 条件 | `Condition` / `AICastCondition` | 释放条件 / AI 自动施法条件 |
| 参数 | `SkillTagsList[{SkillTagConfigID, Value}]` | 技能参数列表 |
| 参数-伤害 | `SkillDamageTagsList[]` | 伤害值参数列表（影响数值表） |
| 提示 | `SkillTipsConditionSkillTagsList[]` | 文案条件参数 |
| AI-智能 | `SmartCastTargetBasePriority` / `SmartCastTargetCondTemplate` / `SmartCastTargetMonsterRankCond[]` / `SmartCastTargetCampCond[]` | AI 选目标基础优先级（距离/血量）/ 模板条件 / 怪物 Rank 条件 / 阵营条件 |
| AI-无目标 | `SmartCastNoTargetIndicatorPos` | 无目标时指示器位置 |
| 释放目标 | `CastTargetCondTemplate` / `CastTargetMonsterRankCond[]` / `CastTargetCampCond[]` | 玩家释放时的目标限制 |
| 指示器 | `SkillIndicatorType` / `SkillIndicatorParam[]` / `SkillIndicatorParamTagConfigIds[]` | 释放指示器类型 / 静态参数 / 动态参数 |
| 品质 | `SkillQuality` / `SkillProperty` (位组) | 品质等级 / 属性位（如"无视禁止施法"） |
| 被动开关 | `IsPassiveSkillHideCD` / `IsPassiveSkillNotRunByCD` | 被动隐藏 CD / 被动不走 CD |
| 连招 | `UseSkillForbidUpdateFaceDir` / `UseType` / `ButtonUpConfig` / `InterruptConfig` / `ReActiveConfig` / `EnhanceSkillBuffConfigID` | 连招期间不更新朝向 / 使用方式 / 抬手配置 / 中断配置 / 重复激活 / 强化 buff |

**保存校验（OnSaveCheck）**：
- 必填 SkillMainType/SkillSubType
- 招式（TBSST_ZHAO_SHI）禁止 CD=0
- 时间约束：`SkillCastFrame ≤ SkillBufferStartFrame ≤ SkillBaseDuration ≤ CdTime`
- CdType=连招时，最后段 BaseDuration 必须为 0（自动修正）
- AISkillRange 必填
- AI 标签"解控"必须配 SkillProperty 的"无视禁止施法"位

**自动化**：保存时自动给 SkillDescEditor 末尾补"。"，调用 `Config.UpdateIndicatorTypeConfig()` 同步指示器配置。

**多等级数据预览**：保存时查 `SkillValueConfig`（按 ID + level 1~30）和 `SkillDescConfig`，在 Inspector 中以 TableList 展示。

### 1.2 SkillTagsConfigNode（技能参数声明）

**对应表**：`SkillTagsConfig`。

**字段表**：

| 字段 | 类型 | 含义 |
|------|------|------|
| `ID` | int | 唯一标识 |
| `Desc` | string | **必填**，编辑器内显示用，节点拷贝后会自动加 `_[ID]` 后缀防重名 |
| `TagType` | TSkillTagType 枚举 | 标签分类（普通参数/AI 参数/伤害参数等） |
| `NameKey` | string | 多语言名称 |
| `DefaultValue` | int | 初始值 |
| `FinalValueEffectID` | SkillEffectConfig ID | 取最终值时执行的效果（用于值的动态计算） |
| `RetainWhenDie` | bool | 死亡时是否保留 |

**作用**：声明运行时挂在战斗单位上的"技能参数"——key 由 (技能ID, TagType, TagID) 三元组决定，value 是 int。被 TSET_GET/SET/MODIFY/ADD_SKILL_TAG_VALUE 系列读写。

**保存校验**：Desc 不能为空，不能保留 `_[ID]` 模板后缀。每次 OnConfigChanged 实时同步到 DesignTable。

### 1.3 SkillSelectConfigNode（筛选根）

**对应表**：`SkillSelectConfig`。`SkillSelectType` 决定具体筛选形态（详见第 3 节）。

**核心字段**：

| 字段 | 含义 |
|------|------|
| `SkillSelectType` | 筛选类型（圆形/扇形/矩形/特定单位/模板...） |
| `Params: List<TParam>` | 参数列表，含义因 SkillSelectType 而异 |
| `SpecialSkillSelectFlag` | 阵营快捷过滤（默认 `TSSSF_Common_DiJun` 敌军通用） |
| `EntityTypeFilters` | 实体类型过滤（默认 `[TENT_BATTLE_UNIT, TENT_SCENE_OBJECT]`） |

**节点显示**：自动名 = `中文枚举描述 [name:ID]`，`isRenamable=false` 不允许手改。

---

### 1.4 SkillEffectConfigNode（效果根）

**对应表**：`SkillEffectConfig`。`SkillEffectType` 决定具体效果（详见第 2 节）。

**核心字段**：

| 字段 | 含义 |
|------|------|
| `SkillEffectType` | 效果类型枚举（约 250 种） |
| `Params: List<TParam>` | 参数列表，含义因 SkillEffectType 而异 |

**所有 TSET_* 都继承它**。Params 数组的语义由 SkillEffectType 决定，**不要混用**。

---

### 1.5 SkillConditionConfigNode（条件根）

**对应表**：`SkillConditionConfig`。`SkillConditionType` 决定具体条件（详见第 4 节）。

**核心字段**：

| 字段 | 含义 |
|------|------|
| `SkillConditionType` | 条件类型枚举（约 95 种） |
| `Params: List<TParam>` | 参数列表 |

---

### 1.6 SkillInterruptConfigNode（中断包）

**对应表**：`SkillInterruptConfig`。结构最简——一个 SkillTagID 引用作为"中断信号"。

**字段**：

| 字段 | 含义 |
|------|------|
| `SkillTagID` (TParam → SkillTagsConfig) | 中断标记，外部通过修改这个 Tag 的值发送中断信号 |

**用途**：在 `TSET_DELAY_EXECUTE` / `TSET_REPEAT_EXECUTE` 中作为中断条件——当指定 Tag 的值变化时，停止延迟/重复任务。

## 2. 效果子节点（TSET_*）

> **公共结构**：所有 TSET_* 都通过 `Params: List<TParam>` 传参。下表 `Params[N]` 的"类型"是该参数的语义类型（`Frame` 帧数 / `‱` 万分比 / `<TableName>` 表格 ID 引用 / `<EnumName>` 枚举 / `bool` / `int` 等）。
>
> **TParam.ParamType（PT）速查**（详见 [[技能编辑器架构#TParam 三元组]]）：
> - PT=0 (TPT_NULL)：普通值
> - PT=1 (TPT_ATTR)：Value 是 `TBattleNatureEnum`（属性 ID，运行时读取单位身上该属性的当前值；例如 91=面向、59=位置X、60=位置Y）
> - PT=2 (TPT_FUNCTION_RETURN)：Value 是另一份 `SkillEffectConfig` ID，**取它的返回值**
> - PT=3 (TPT_SKILL_PARAM)：Value 是 `SkillTagsConfig` ID
> - PT=5 (TPT_COMMON_PARAM)：Value 是 `TCommonParamType` 枚举值，**指向运行时上下文**（V=1 主体单位实例ID / 2 目标单位实例ID / 3 施法者实例ID / 4 主体伤害归属单位 等）
>
> 例：`{V:1, PT:5}` = "运行时主体单位实例ID"（即施法者本人），不是"槽位 1"也不是"BattleCustomParamConfig.ID=1"。

### 2.1 流程控制类（最高频）

#### TSET_ORDER_EXECUTE — 顺序执行
**用途**：同步执行多个子效果（一帧内全部触发）。
**Params**：
- `Params[0..N]` = `<SkillEffectConfig>` 子效果 ID 列表（可变长）

**典型用法**：技能拆成"播动作 + 创建特效 + 造伤害"三步时拼装。

---

#### TSET_DELAY_EXECUTE — 延迟执行
**用途**：延迟若干帧后执行单个子效果。

**Params**：
- `Params[0]` = 延迟帧数 (Frame)
- `Params[1]` = `<SkillEffectConfig>` 子效果 ID
- `Params[2]` = 死亡时是否继续 (bool 0/1)
- `Params[3]` = `<SkillSelectConfig>` 筛选 ID（可选，决定执行作用对象）
- `Params[4]` = `<SkillInterruptConfig>` 中断配置 ID（可选）
- `Params[5]` = 保留
- `Params[6]` = 急速影响 (bool 0/1，是否被攻速等急速属性影响延迟时长)

**易错**：忘记急速影响会导致连招手感不一致。

---

#### TSET_REPEAT_EXECUTE — 重复执行
**用途**：按间隔重复执行子效果 N 次。

**Params**：
- `Params[0]` = 间隔帧 (Frame)
- `Params[1]` = 次数 (int)
- `Params[2]` = 是否立刻执行第一次 (bool)
- `Params[3]` = `<SkillEffectConfig>` 子效果 ID
- `Params[4]` = `<SkillSelectConfig>` 筛选 ID
- `Params[5]` = `<SkillInterruptConfig>` 中断 ID
- `Params[6]` = 死亡继续 (bool)
- `Params[7]` = `<SkillEffectConfig>` 结束效果 ID（重复完后执行）
- `Params[8]` = 战斗结束继续 (bool)
- `Params[9]` = 急速影响 (bool)

**保存校验**：间隔=0 且次数>100 直接报错（防死循环）。

---

#### TSET_SWITCH_EXECUTE — Switch 分支
**用途**：按值匹配执行不同子效果（最多 10 组 case）。

**Params**：
- `Params[0]` = Switch 取值（int 或 引用其他 TSET 输出）
- `Params[1]` = `<SkillEffectConfig>` Default 子效果 ID
- `Params[2k+2]` = Case k 匹配值
- `Params[2k+3]` = `<SkillEffectConfig>` Case k 子效果 ID
- 其中 k = 0, 1, 2, ..., 9

**自动化**：保存时 `RemoveInvalidPatam` 自动清理空 case 对（值=0 且效果=0 的尾部）。

**典型用法**：根据角色阶段、buff 层数、五行属性等切换不同行为。

---

#### TSET_RUN_SKILL_EFFECT_TEMPLATE — 模板技能效果
**用途**：跳到外部模板执行（**最重要的复用机制**）。

**Params**：
- `Params[0]` = 主体单位 ID（PT=5 表示槽位/上下文）
- `Params[1]` = `<SkillSelectConfig>` 目标筛选单位 ID
- `Params[2]` = `<SkillEffectConfig>` 模板 SkillEffectID（指向技能模板内的根节点）
- `Params[3..]` = 模板自定义参数（按模板定义动态数量）

**特殊字段**：`TemplateData.TemplatePath` = 模板 JSON 的相对路径。

**保存校验**：检测 TemplateID 与 Params 中的实际值一致。

**注意**：可层层嵌套——伤害流程模板自己又会引用"是否暴击检查"等子模板。

---

#### TSET_PROBABILITY_EXECUTE — 概率执行
**用途**：按万分比概率走成功/失败分支。

**Params**：
- `Params[0]` = 成功概率 (‱)
- `Params[1]` = `<SkillEffectConfig>` 成功效果 ID
- `Params[2]` = `<SkillEffectConfig>` 失败效果 ID

---

#### TSET_CONDITION_EXECUTE — 条件执行
**用途**：按条件结果走分支。

**Params**：
- `Params[0]` = `<SkillConditionConfig>` 条件 ID
- `Params[1]` = `<SkillEffectConfig>` 满足时执行
- `Params[2]` = `<SkillEffectConfig>` 不满足时执行

---

#### TSET_TERMINATE_TASK — 终止执行
**用途**：取消已注册的延迟/重复任务（与 DELAY/REPEAT 配合使用）。

---

### 2.2 事件机制类

#### TSET_REGISTER_SKILL_EVENT — 注册技能消息
**用途**：监听特定技能事件，触发时执行子效果。

**Params**：
- `Params[0]` = `<SkillEventConfig>` SkillEventID
- `Params[1]` = `<SkillEffectConfig>` 绑定的技能效果 ID
- `Params[2]` = `<SkillConfig>` 绑定的技能 ID
- `Params[3]` = `<SkillSelectConfig>` 筛选 ID
- `Params[4]` = `<SkillConditionConfig>` 条件 ID
- `Params[5]` = 触发次数限制 (int, 0 = 无限)
- `Params[6]` = 目标阵营过滤
- `Params[7]` = 消息筛选-技能 ID
- `Params[8]` = 消息筛选-子类型
- `Params[9]` = 事件子类型 (TSkillEventSubType 鉴别器)
- `Params[10]` = 事件子类型值（最后一参数下拉数据源由 Params[9] 决定：属性/技能槽/技能ID/状态/单位子类型/操作类型/单位类型）

**易错**：Params[9] 切换时 Params[10] 的下拉源会变，**改完要重新选**。

---

#### TSET_UNREGISTER_SKILL_EVENT — 反注册技能消息
**用途**：取消监听（与 REGISTER 配对使用）。

**Params**：与 REGISTER 的 [0,1,2,9,10] 对应。

**保存校验**：与对应 register 节点参数一致性检查。

---

#### TSET_FIRE_SKILL_EVENT — 发送技能消息
**用途**：主动触发事件（让监听者感知）。

**Params**：
- `Params[0]` = 发送者 ID
- `Params[1]` = `<SkillEventConfig>` SkillEventID
- `Params[2..13]` = 事件参数 1~12（参数显示名按 SkillEventConfig.ParamName1~N 动态命名）

- `Params[11]` = 纯特效（不参与逻辑） (bool)

**Params**：

- `Params[1]` = 阵营
- `Params[2]` = 玩家索引
- `Params[3]` = 出生前 `<SkillEffectConfig>` 效果

### 2.5 技能参数读写

#### TSET_GET_SKILL_TAG_VALUE — 获取技能参数值
**Params**：
- `Params[0]` = 拥有者单位
- `Params[1]` = 技能 ID（`>0` 表示具体 ID, `<0` 表示槽位/实例）
- `Params[2]` = `<SkillTagsConfig>` SkillTagsConfigID（VD_TagsValue 下拉）

### 4.5 TSCT_VALUE_COMPARE — 数值比较（schema v3 终版 / 2026-05-13 入库）

> 来源：[memory/feedback_value_compare_schema.md](../../../memory/feedback_value_compare_schema.md) / 2026-05-12 配 30212018 时连续 3 版踩错才校正。

**Params 顺序铁律**（V3 终版）：

| P# | 含义 | ParamType | 值 / 示例 |
|----|------|-----------|-----------|
| `P[0]` | 被比较值 NodeRef | **PT=2** (NodeRef) | `V=某 NUM_CALC 节点 ID` |
| `P[1]` | 比较操作符 op | PT=0 | **`1 = ==`** ⚠️ |
| `P[2]` | target value | PT=0 | 常量数字 |

⚠️ **反直觉**：P[1]=op P[2]=target（**不是** P[1]=target P[2]=op）。op=1 才是 ==（**不是** op=0）。

**op 值速查**：

| op 值 | 含义 |
|-------|------|
| **1** | == |
| **4** | ≥（>=） |
| 其他 | 待源码验证 |

**金标参考样本（30212010）**：
- `320359` Desc "判断传入数据 == 0"：`P[0]=NodeRef(32003040, PT=2) / P[1]=1 / P[2]=0`
- `320355` Desc "本关卡 == 1"：`P[0]=NodeRef / P[1]=1 / P[2]=1`

**Sensor**：节点"判断永远 true / 永远 false" → 第一时间 check Params 顺序。

**配套范式**：REPEAT body 内 "ADD-then-CONDITION"（详见 [SkillEvent系统.md §条件类节点 schema](../mental_model/SkillEvent系统.md)）。

## 易错点全列表

1. **ID 重复**：跨文件出现相同 `SkillEffectConfig_<ID>` 报错。新建必须从 `SvnIgnore/LocalID.json` 分配。
2. **节点拷贝重名**：SkillTagsConfigNode 拷贝后 Desc 自动加 `_[ID]`，**保存前必须改回**。
3. **CdType=连招最后段 BaseDuration 必须 0**：自动修正，但会报警。
4. **AI"解控"必须配 SkillProperty 位**：否则保存校验失败。
5. **筛选默认敌军通用**：配友军/特定单位需手改 `SpecialSkillSelectFlag`。
6. **TSET_REPEAT_EXECUTE 间隔=0+次数>100 直接报错**：防死循环。
7. **TParam.ParamType=5 是 TCommonParamType（上下文常量）**：V=1 主体单位 / V=2 目标单位 / V=3 施法者 / V=4 主体伤害归属单位 …（共 46 项），不是"槽位"也不是百分比。
8. **TSET_REGISTER_SKILL_EVENT Params[10] 联动**：Params[9] 切换时 Params[10] 下拉源会变，**改完要重新选**。
9. **TSET_NUM_CALCULATE 末尾空对自动清理**：保存时删空（运算符=0 数值=0）的尾部，**顺序混乱时易误删**。
10. **TSET_RUN_SKILL_EFFECT_TEMPLATE 模板 ID 校验**：保存时检测 TemplateID 与 Params 中实际值一致，不一致报错。
11. **RefConfigBaseNode 不能作为模板的第一个节点**：会报错，第一个节点应该是常规节点。
12. **JSON 出现 `.json.mine`**：SVN 冲突未解决，编辑器强制弹窗。
13. **TSET_DELAY_EXECUTE 急速影响默认开**：连招手感不一致时检查这一参数。
14. **TSET_CREATE_BULLET BulletID=0 报错**：必须显式指定。
15. **TSET_USE_SKILL 已废弃**：用 TSET_AI_USE_SKILL 替代。
16. **TSET_MODIFY_ENTITY_STATE 慎用**：会顶替其他状态，建议用 buff。
17. **TSET_ADD_ENTITY_STATE 恢复填 -1**：不是 0。
18. **筛选节点 EntityTypeFilters 默认有 SCENE_OBJECT**：场景物会被纳入筛选，特殊场景需要排除。
