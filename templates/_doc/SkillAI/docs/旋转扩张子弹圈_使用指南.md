# 旋转扩张子弹圈模板使用指南（v1.0）

## 这个模板做什么

发射 N 颗独立子弹，子弹按圆周均匀分布，然后整团绕主角旋转并径向扩张或收缩。

**模板根 ID**：32300001  
**OnTick 入口 effect ID**：32300101

## 设计边界

模板只负责几何运动：

- 子弹数、初始半径、切线偏移角
- 径向速度、径向加速度、角速度
- 最大/最小半径销毁
- 圆心是否跟随主角

以下内容不在模板里暴露参数，统一由 `BulletConfig` 或它的命中链路负责：

- `lifetime`：用 `BulletConfig.LastTime`
- `damagePerHit`：用子弹命中链路和技能伤害 tag
- `pierceCount`：用 BulletConfig / 命中链配置

## BulletConfig Contract

调用方传入的 `bulletConfig` 必须是真实存在的 `BulletConfig.ID`，并且这个 BulletConfig 需要配置：

```text
AfterBornSkillEffectExecuteInfo.SkillEffectConfigID = 32300101
```

否则子弹可以被创建，但不会进入旋转扩张的 OnTick 更新链。

推荐 BulletConfig 运动字段：

| 字段 | 建议 |
|---|---|
| `Speed` | 0，让模板接管位移 |
| `FlyType` | 0 或不会抢位移控制的类型 |
| `TracePathType` | 0 |
| `LastTime` | 由技能需求决定，作为兜底寿命 |
| `AfterBornSkillEffectExecuteInfo.SkillEffectConfigID` | 32300101 |

## 10 项模板参数

| # | 参数 | 默认 | 单位 | 说明 |
|---|---|---:|---|---|
| 1 | `bulletCount` 子弹数N | 8 | 颗 | 同帧生成数量；建议 <= 50 |
| 2 | `initialRadius` 初始半径R0 | 200 | 单位 | 1 米 = 100 单位 |
| 3 | `tangentOffsetAngle` 切线偏移角 | 0 | 度 | 0=沿切线，90=沿径向 |
| 4 | `radialVelocity` 径向速度 | 10 | 单位/帧 | 正外扩，负内收 |
| 5 | `radialAcceleration` 径向加速度 | 0 | 单位/秒² | 每帧增量为 aR/30 |
| 6 | `angularVelocity` 角速度 | 3 | 度/帧 | 正逆时针，负顺时针 |
| 7 | `maxRadius` 最大半径 | 1500 | 单位 | R 超过后销毁该子弹 |
| 8 | `minRadius` 最小半径 | 0 | 单位 | R 低于后销毁该子弹 |
| 9 | `followPlayer` 是否跟随主角 | 0 | 0/1 | 0=锁定释放瞬间圆心，1=逐帧跟随 |
| 10 | `bulletConfig` 子弹ID | 0 | BulletConfig.ID | 必填，且 AfterBorn=32300101 |

## 推荐预设

| 预设 | N | R0 | vR | aR | φ | ω | follow |
|---|---:|---:|---:|---:|---:|---:|---:|
| 花瓣绽放 | 8 | 50 | 5 | 0 | 0 | 3 | 0 |
| 螺旋扩散 | 8 | 30 | 8 | 0 | 0 | 12 | 0 |
| 向心收缩 | 8 | 600 | -7 | 0 | 0 | -4 | 0 |
| 风暴漩涡 | 8 | 80 | 3 | 0 | 30 | 18 | 0 |
| 爆裂冲击波 | 8 | 0 | 0 | 1500 | 0 | 1 | 0 |
| 先扩后收 | 8 | 100 | 13 | -10 | 0 | 2 | 0 |

换算：

- 距离：米 × 100 = 项目单位
- 速度：米/秒 × 100 ÷ 30 = 单位/帧
- 角速度：度/秒 ÷ 30 = 度/帧

## 常见问题

**子弹生成但不旋转扩张**  
检查传入的 BulletConfig 是否把 `AfterBornSkillEffectExecuteInfo.SkillEffectConfigID` 配成 `32300101`。

**报 `RepeatExecute Execute Count Too Big`**  
`bulletCount` 太大。保守建议 `bulletCount <= 50`。

**子弹运动很怪**  
检查 BulletConfig 是否还有 `Speed / FlyType / TracePathType` 等自带运动逻辑在和模板抢位移控制。
