# Sticky Note 写作模板（团队统一规范 · v4）

## 概述

每个 SkillGraph 蓝图都应在画布上挂一张 Sticky Note。**目标只有一个：让任何人（包括三周后的自己）一眼把这个技能/心法的逻辑读清楚——因果链、条件分支、关键参数。**

> **2026-05-09 v3 → 2026-05-13 v4 重写说明（rule_2 思想史保留）**：
> - **v1 已废**：6 段时间轴 + 双数据卡，过于工程师化、堆数据。
> - **v2 已废**：5 段问答式 + 文学画面感叙述，太文学、逻辑性弱。
> - **v3 已废**：5 段纯逻辑骨架——作用 / 流程 / 特殊条件 / 参数 / 联动。**用户 2026-05-13 反馈"太程序化看不懂"。** v3 骨架（5 段结构）保留有效，仅写法升级。
> - **v4（当前版 / 2026-05-13）**：v3 骨架 + **大白话口语风格**。流程段 + 参数描述全部口语化。来源：[memory/reference_sticky_note_template.md](../../../memory/reference_sticky_note_template.md) / 用户 2026-05-13 拍板。

## v3 → v4 写法对比（核心升级）

| 段 | v3（程序化 / 已废） | v4（大白话 / 当前） |
|----|---------------------|---------------------|
| **流程** | "1. 释放 → 生成 3 颗子弹" | "1. 你按技能键，前面噗噗噗冒出 3 道剑气" |
| **参数** | "子弹数量 = 3" | "改这里 → 子弹数量。写 3 就 3 颗，写 5 就 5 颗" |
| **节点 Desc** | "ADD_BUFF duration=3" | "贴个持续掉血的标记 / 挂 3 秒" |

**铁律**：
- 口语词（"你按键""倒霉蛋""贴标记""噗噗噗冒出"），**不用**编辑器术语（BulletConfig/SkillTag ID 永远不出现）
- **关键数值必须保留**（不能因为"大白话"漏数字）
- 参数描述说"改这里 → 影响什么"，策划读完知道怎么改

## 设计原则

| 原则 | 说明 |
|------|------|
| 逻辑清晰 | 因果链（→）+ 条件分支（满足 X 时 Y）+ 模块边界清晰 |
| 不文学 ⚠️ v4 调整 | v3 完全禁文学；**v4 允许少量画面感口语**（"噗噗噗冒出""倒霉蛋""贴标记"），但**不用比喻 / 不用情绪词** |
| 数据归位 | 流程段不带数值（但**关键结构数值如"3 道剑气"保留**），所有**可调**数据集中在参数段 |
| 不堆术语 | 不写编辑器术语（TSET/ORDER/TableTash）、节点 ID、Tag ID |
| 母模板复用 | 技能/心法/被动共用 5 段，无关的写"—" |
| 不写改动日志 | 旧 vs 新的话彻底剔除（→ git / PostMortem） |
| **大白话** ⭐ v4 新增 | 流程 + 参数描述全部口语化 / 策划读完知道"改这里 → 影响什么" |

## 字段说明

### 顶部身份证

`【技能名】 8位ID  宗门·品阶·类型`

- 技能名 / 8 位 ID / 宗门 / 品阶 / 类型（主动技能 / 被动技能 / 心法 / 升阶技能）

### 【作用】

**一句话定位**。说类型 + 解决场景。**不重复**身份证里的内容。

✅ "木宗入门主动，正面扇形 AOE。"
❌ "千叶散华是木宗门人阶主动技能。"（重复身份证）
❌ "TSET_RUN_SKILL_EFFECT_TEMPLATE 调扇形模板。"（编辑器术语）

### 【流程】（核心段）

**因果链 + 大白话**。用编号步骤 + `→` 表示因果。**只讲"发生什么"，不带任何可调数值**。**v4 改为口语化叙述**。

写作要点：
- 每步用 `<触发> → <结果>` 格式
- **v4 改用大白话**：你按键 / 倒霉蛋被打 / 贴标记 / 飞回来 / 噗噗噗冒出
- 形态结构性事实可以保留（"30 颗叶刃""3 圈层"——这是技能形态本身，不是可调参数）
- 时间长度、数值伤害、CD、范围 全部移到参数段
- 步骤数量按技能复杂度，3~5 步内最佳

**v4 好示例**（口语化 + 因果清晰）：
- ✅ "1. 你按技能键 → 抬手不能动，不能被打断"
- ✅ "2. 抬手结束 → 前面噗噗噗冒出 3 道剑气"
- ✅ "3. 剑气飞 8 米打中倒霉蛋 → 爆一下 + 贴个持续掉血的标记"

**v3 旧示例**（程序化 / v4 已废）：
- ⚠️ "1. 按键 → 抬手硬直，期间不可移动·不可被打断。"（v3 当时认为是好示例）

**坏示例（永远不要写）**：
- ❌ "1. 按键 → 抬手 20 帧 (0.67s) 硬直。"（数值要去参数段）
- ❌ "1. 抬手轻挥，绿色叶刃从指尖绽散。"（文学修辞）
- ❌ "1. 调用 TSET_RUN_SKILL_EFFECT_TEMPLATE 启动持续施法。"（编辑器术语）

### 【特殊条件】

**分支 / 强化 / 限制条件**。用 `条件 → 效果` 格式。

适合写：
- 多段伤害分支（"第 3 次伤害翻倍"）
- 距离衰减（"距离 < 3 米时 ×1.5"）
- buff 强化（"带 X buff 时改为穿透"）
- 释放限制（"只能在战斗状态使用"）

**没有特殊条件就写 "—"**。不要硬凑。

### 【参数】

**所有可调数值集中在这里 + v4 大白话说"改这里 → 影响什么"**。3~6 行，按字段类型组织：

```
范围：<距离 / 形状 / 最大射程>
节奏：<抬手 Xs｜CD Xs｜灵力 X｜MP X>
伤害：<普通版公式 ／ 强化版公式>
<其他：持续时间、堆叠层数等>
```

**v4 大白话风格**（每个参数说"改这里 → 影响什么"）：
- ✅ "**子弹数量** → 写 3 就 3 颗，写 5 就 5 颗"
- ✅ "**飞行距离** → 改成 100 = 飞 1 格远，改成 800 = 飞 8 格远"
- ✅ "**抬手** → 0.67 秒（20 帧）；改大 → 起手更慢但伤害更高"
- ⚠️ "子弹数量 = 3"（v3 程序化 / 已废）

**不写**：Tag ID / Config ID / 节点 ID / 模板内部参数（baseDist / layerStep 等）。

### 【联动】

**与其他技能/buff/心法的关联**。没有写 `—`。

常见入选：
- 跨技能共享计数池 / 写读其他技能 buff
- 与某技能互斥 / 组合限制
- 受某心法增强
- 触发某 buff 后改变行为

## v4 套路速查（每段大白话怎么写）

| 段 | v4 大白话套路 | 实例 |
|----|--------------|------|
| 【作用】 | "X 宗门 X 类型，干 X 用" | "木宗入门主动，正面砸一片" |
| 【流程】 | "你 X / 倒霉蛋 Y" 第二人称口语 + 拟声词 | "你按键 → 前面噗噗噗冒出 3 道剑气" |
| 【特殊条件】 | "撞上 X 就 Y" 强化分支用口语 | "倒霉蛋身上带毒标记 → 这一击多打 50%" |
| 【参数】 | "改这里 → 影响什么" 二元结构 | "**伤害** → 改 6000 = 60%攻；改 10000 = 100%攻" |
| 【联动】 | "跟 X 配合 / 给 X 喂状态" | "每次打中 → 给三重碧叶喂 +1 计数" |

**口语词速查（鼓励使用）**：
- 你 / 你按键 / 你这边
- 倒霉蛋 / 那个敌人 / 那家伙
- 噗噗噗 / 嗖一下 / 砰一下
- 贴标记 / 挂状态 / 喂层数
- 多打 X% / 少打 X% / 翻倍
- 飞回来 / 砸地上 / 冒出来

**编辑器术语禁忌（永远不写）**：
- BulletConfig / SkillTag / SkillEffect / TSET_xxx / ORDER / REPEAT
- Tag ID / Config ID / 节点 ID
- "调用模板"/"触发 effect"/"挂 buff"（→ 改"砸一片"/"贴标记"）

## 真实样本：千叶散华 (30212009) — v4 大白话版

```
【千叶散华】 30212009  木宗门·人阶·主动

【作用】
木宗入门主动，正面砸一片。

【流程】
1. 你按技能键 → 抬手不能动，不能被打断
2. 抬手结束 → 身前扇形喷出 30 颗叶刃，分 3 圈层依次飞出
3. 叶刃打中倒霉蛋 → 砸一下造成伤害

【特殊条件】
- 每三次施展为一轮
- 同时满足两条 → 伤害翻倍：
  ① 你身上挂着「三重碧叶」标记
  ② 这一发是本轮第 3 次

【参数】
范围：身前 90° 扇形，最大射程 8 米
节奏：抬手 0.67s（20 帧）｜CD 4.8s｜灵力 6
**伤害**：普通版 60%攻 + 1000 ／ 强化版 100%攻 + 2000

【联动】
- 每次释放 +1 喂给跨技能 Tag 池，跟「碧叶」系列共享
```

## 心法 / 被动套用

5 段照写，语义按下表对应：

| 段 | 心法/被动写什么 |
|----|----------------|
| 【作用】 | 一句话定位（"X 心法被动，X 时机 X 效果"） |
| 【流程】 | 触发条件 → 生效内容 → 收尾。如"受击 → 反伤 → 持续 Xs"。心法被动也走因果链 |
| 【特殊条件】 | 触发概率 / 堆叠层数 / 前置 buff / 升阶质变条件 |
| 【参数】 | 触发概率 / 冷却 / 持续时长 / 强度数值 |
| 【联动】 | 与其他技能/buff/心法的关联 |
